## 2. 小工单拆分

### CM-0：右键菜单现状审查与精确边界

类型：只读审查。

目标：

- 定位主界面右键区域。
- 定位周界面白色空白区域与日期映射。
- 定位顶部按键组现有动作入口。
- 定位已有菜单组件和可复用 icon。
- 输出 CM-1 / CM-2 / CM-3 的精确实现建议。

允许修改：

- `manage_instruction/Work_Log.md`
- `manage_instruction/Work_Task_Prompts.md`（仅在需要维护复核锚点时）

禁止修改：

- `src/`
- `assets/`
- `main.py`
- `requirements.txt`
- `schedule.db`

### CM-1：公共 icon-text 上下文菜单组件试点

类型：低风险组件提取。

目标：

- 根据 `CM-0` 结论，新增或最小复用/扩展公共菜单组件。
- 若现有 `SharedMoreMenu` / `ScheduleContextMenu` 不适合作为通用右键菜单，再新增 `src/ui/common/action_context_menu.py`。
- 菜单项为左图标、右文字。
- 支持视图子菜单在右侧紧邻弹出。
- 仅做组件 import / offscreen 验证，不接真实业务 UI。

候选文件：

- `src/ui/common/action_context_menu.py`

禁止事项：

- 不接入 `MainWindow`。
- 不接入 `WeekWindow`。
- 不改顶部按钮组。
- 不改已有菜单组件。

### CM-2：主界面日程区域右键菜单接入

类型：单点 UI 接入。

目标：

- 在主界面日程卡片区、空状态区或日程列表容器区右键弹出菜单。
- `添加` 复用 `MainWindow.switch_to_add_page()`。
- `视图` 子菜单复用 `MainWindow.switch_view(...)`。
- 四象限保持禁用或占位提示。

禁止事项：

- 不改数据库。
- 不改 picker。
- 不改详情弹窗。
- 不实现排序 / 筛选 / 换肤。
- 不改变顶部按钮组行为。

### CM-3：周界面日期空白区域右键菜单接入

类型：单点 UI 接入。

目标：

- 在周界面某日期白色空白区域右键弹出菜单。
- `添加` 使用右键所在日期。
- 过去日期不能添加日程。
- `视图` 子菜单复用已有视图切换链路。

禁止事项：

- 不改变卡片拖拽排序。
- 不改变卡片右键菜单。
- 不改变单击选中日期。
- 不改变双击跳转日视图。
- 不改数据库写入逻辑。

### CM-4：右键菜单整体验收

类型：验收与归档准备。

目标：

- 验证主界面右键菜单。
- 验证周界面右键菜单。
- 验证顶部按钮组旧行为。
- 验证路由、添加、过去日期限制。
- 确认数据库无非预期 tracked diff。
