## 4. 关键规则

日程圆点：

- 无日程日期不显示圆点。
- 今天及未来只统计未完成日程。
- 未来未完成日程最高紧急性：高为红色，中为黄色，低为绿色。
- 今天及未来没有未完成日程则不显示圆点。
- 过去日期有日程且全部完成为白色圆点。
- 过去日期有日程且存在未完成为灰色圆点。
- 过去日期无日程不显示圆点。

日期交互：

- 今天数字金色，不代表选中。
- 单击日期选中。
- 双击日期跳日视图。
- 双击跳转时关闭所有月界面持久浮窗。

浮窗：

- hover 预览为只读，鼠标移走立即隐藏。
- 单击日期持久浮窗显示在月界面外侧，类比待办面板。
- 持久浮窗可多个并存。
- 月界面关闭、隐藏或跳转日视图时必须关闭全部持久浮窗。

右键菜单：

- 复用 `ActionContextMenu`。
- 右键日期作为菜单上下文。
- 右键建议不改变当前选中日期。
- 添加使用右键日期。
- 过去日期添加禁用。
- `month` 当前项和周视图日期上下文是否处理，需 `M-0` 先确认。
- `M-6` 推迟到 M-5 后续添加表单能力补齐之后，右键“添加”复用补齐后的同一入口。

月界面添加表单：

- 左侧按键组和搜索框下方区域，类比主界面按键组下方的添加区域。
- 月界面可做竖向压缩适配，不必完全复制主界面 UI。
- 详情输入框可默认显示。
- picker 承接方式必须先审查确认，不得默认照搬 `MainWindow` 页面栈。
- `repeat_rule` 的真实入库值、显示值和 `ScheduleRepeatService` 归一化规则必须先审查确认；后续不新增新规则，只沿用现有真实值体系。
- 保存后必须刷新月格 marker cache、hover 预览数据和已打开的同日期持久 panel；如暂时不能刷新全部 panel，至少关闭或重载目标日期 panel。
