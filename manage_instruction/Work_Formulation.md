# 当前阶段：功能补充 - 月界面优先

## 1. 阶段背景

架构改写主线已经结束。当前项目进入功能补充阶段。

当前不优先启动四象限视图。原因：

- 月界面功能仍未完善。
- 月界面属于现有功能体系的一部分，优先补齐可降低整体使用断层。
- 四象限属于新增功能，应在现有日/周/月/待办基础稳定后再规划。

---

## 2. 当前阶段目标

优先完善月界面功能，并补充少量低风险跨视图交互增强。

近期目标：

- 建立月界面当前行为和缺口地图。
- 补齐月界面 MVP 能力。
- 明确月界面与日视图、周视图、添加页、详情弹窗、刷新链路的关系。
- 审核并实现低风险交互增强，例如周界面日期双击跳转到对应日视图。
- 月界面稳定后，再制定后续功能补充规划。

- 不做完整云同步。

## 4. 建议小工单顺序

### M-0：月界面现状审查与功能缺口地图

只读审查：

- `src/ui/month_window.py`
- `src/ui/main_window.py`
- `src/ui/dashboard.py`
- `src/ui/week_window.py`
- `src/ui/calendar_pop.py`
- `src/ui/add_view.py`

输出：

- 月界面已实现功能。
- 月界面未完成或行为不明确功能。
- 与日/周/待办的交互差异。
- 后续 M-1/M-2/M-3 建议。

### W-1：周界面日期双击跳转日视图

低风险增强：

- 单击日期仍保留为周界面选中日期。
- 双击日期跳转到对应日期的日视图。
- 复用现有主窗口跳转日期逻辑。

### M-1：月界面 MVP 功能补齐

候选能力：

- 点击日期后查看对应日期日程。
- 从月界面添加该日期日程。
- 与主窗口日视图跳转保持一致。
- 保存后刷新相关视图。

### M-2：月界面交互增强

候选能力：

- 今日和选中日期视觉状态。
- 日期日程数量或状态提示。
- 跨月切换行为整理。
- 月界面挂起/恢复回归。

### M-3：月界面整体验收

验证：

- 日/周/月/待办切换。
- 添加、编辑、删除后刷新。
- picker 返回。
- 详情弹窗回流。
- 数据库无异常改动。

---

## 5. 执行策略

- 每个小工单只允许小范围修改。
- 涉及 UI 行为变化时，必须先记录旧行为。
- 涉及跨视图跳转时，优先复用已有方法，不复制逻辑。
- 涉及数据库写入时，必须明确是否会改 `schedule.db`，并在验收后确认无非预期 tracked diff。
- 每个执行任务完成后等待主窗口复核，不直接提交 Git。
