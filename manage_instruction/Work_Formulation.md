# Work Formulation

用途：记录当前阶段的总体规划、目标边界和小工单路线。由主窗口在审核决策窗口方案后维护。

---

# 当前状态：搜索 / 筛选功能第三轮

当前小工单：将日界面既有搜索与筛选接入课表模式。

目标：

- 卡片模式与课表模式共享同一组筛选基线、搜索临时条件和纯查询服务。
- 日课表保持原有当前日期与跨午夜显示数据来源，只在交给绘制器前执行内存查询。
- 搜索与筛选结果为空时保留课表时间网格，不切换为卡片空状态。
- 课表颜色、详情、右键、拖拽、拉伸和日期滚动逻辑保持不变。

已归档阶段：

- 月界面功能补齐阶段。
- 月日程持久 panel 详情与编辑路由阶段（`MP-0 ~ MP-5`）。
- 坐标看板与主题选色弹窗第一版。
- 坐标看板归档后增强与高 DPI / 多屏回归。
- 日 / 周 / 月持久弹窗课程表模式阶段。

---

## 当前边界

- 本轮仅接入日界面课表；周界面、月界面和待办暂不接入。
- 卡片使用当前日期源；课表沿用当前日期加跨午夜所需次日源，逐字符输入不重复扫描数据库。
- 不修改数据库结构，不保存搜索关键词和筛选条件到 `QSettings`。
- 第一轮 `ScheduleSearchPanel` 暂不作为日界面入口，保留给后续“更多 → 全部日程”全局结果窗口复用。
- “全部日程”仍需先设计结果承载页面，本轮不伪实现全局搜索。

---

## 后续候选

1. 用户人工验收日课表逐字符显示、条件筛选和清空恢复行为。
2. 将同一查询模型接入周界面当前周、月界面当前自然月。
3. 设计“全部日程”全局结果窗口，并复用查询选项控件。
4. 导出功能。
5. 自动化测试与 CI 基线。
