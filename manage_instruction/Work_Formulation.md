# 当前状态：整卡滚动与日 / 周卡片多选已完成

本轮仅修正日界面和周界面卡片模式的鼠标滚轮行为。

目标：

1. 鼠标滚轮每一格恰好移动一条卡片。
2. 步长必须包含卡片高度和相邻卡片间隙：日界面 `60 + 8px`，周界面 `46 + 2px`。
3. 每次滚动后，新的第一张卡片必须完整对齐原第一张位置。
4. 可视区底部只显示完整卡片；不足一个步长的余量作为透明留白，不露出半张卡片。
5. 日 / 周课表模式、拖拽排序、查询、刷新和其他视图不变。

完成结果：

- 新增共享 `CardStepScrollArea`，统一维护整卡吸附点、窗口尺寸变化和卡片数量变化。
- 日界面滚动步长为 `68px`，周界面滚动步长为 `48px`。
- 内容溢出时按完整卡片数量缩减 viewport 底部，多余区域保持透明，不绘制下一张残卡。
- 定向 PyQt6 离屏几何回归和相关文件语法检查均已通过。
- 追补修复日界面头部右键菜单：该入口使用独立 `HeaderBar` 菜单构造，现已在日视图卡片模式显示“多选”，课表模式及非日视图不显示。

## 日界面卡片多选

目标：

1. 从日卡片模式右键“多选”进入独立经典弹窗，不占用或移动日程滚动区。
2. 日程前方优先级圆点在多选中改为选择圆：未选中为白色，选中为主题上沿色 `65%` 透明度。
3. 弹窗内容仅一行，依次使用图标提供“全选 / 全不选、完成、撤销、删除、退出”；按钮为方形并使用主题上下沿中值背景。
4. 完成、撤销、删除按当前选择和完成状态动态启用或灰化。
5. 批量状态与删除使用 Repository 原子写入，一次刷新并发出一次全局刷新信号。
6. 弹窗使用主题上下沿渐变，头部以下覆盖 `70%` 白色遮罩；头部左侧标题，右侧紧贴置顶和关闭键。
7. 弹窗置顶默认继承主窗口偏好，但只修改自身置顶状态；关闭弹窗、切换课表或离开日视图时自动退出。

完成结果：

- 多选只作用于当前日卡片模式实际显示的卡片；重复日程保持现有单实例状态 / 删除语义。
- 原日界面固定操作卡及占位已删除；打开多选不会重置滚动位置或改变完整卡片可视数量。
- 多选期间允许继续拖拽日程卡片；弹窗完全独立于列表，不会进入排序提取、分组边界或数据库写回。
- 日卡片拖拽期间暂停界面重绘刷新；若 30 秒自动刷新或查询重绘在 `QDrag.exec()` 内触发，只记录待刷新状态，拖拽结束后再补刷新，避免活动拖拽源被销毁。
- PyQt6 离屏弹窗、按钮状态、置顶切换、关闭退出、滚动保持和临时数据库批量事务回归均通过。

## 周界面卡片多选

目标：

1. 周卡片模式右键“多选”后打开独立经典双行弹窗，恢复周头部原有空白区域。
2. 第一行使用 `Mon / Tue / Wed / Thu / Fri / Sat / Sun / All` 八个圆形透明文本按钮选择全选范围；文字为灰色，选中显示 `2px` 白边，未选中无边框。
3. 第二行与日弹窗一致，使用 `Multiplechoice / all_no、finished、withdraw、delete、exit` 五个方形图标按钮执行全选 / 全不选、完成、撤销、删除和退出。
4. 两行左右边界一致，首个和末个按钮中心分别处于同一竖直线；可用功能图标为白色，无效功能图标为灰色。
5. 七天可多选；七天全选时 `All` 自动选中，切换 `All` 同步七天，移除任意一天后 `All` 取消选中。
6. 周卡片圆点在多选中改为白色 / 主题上沿色 `65%` 透明度的选择圆；完成、撤销和删除按当前选择状态动态启用。

完成结果：

- 多选默认作用范围为周一至周日；第一行只限定“全选 / 全不选”的范围，第二行状态操作处理当前实际选中的卡片。
- 原周头部固定双行菜单及宿主已删除；弹窗与周列表、头部导航和窗口拖拽互不占位。
- 选择集合使用日程 ID 保存，查询、排序、刷新和跨列拖拽后与当前周实际卡片重新同步。
- 多选期间保留周卡片原有同列 / 跨日期拖拽；单击卡片主体不打开详情，双击仍可打开详情。
- 切换课表、离开周视图或进入添加页时自动退出周多选。
- 真实周窗口及独立弹窗离屏布局、8 / 5 按钮端点对齐、日期联动、图标切换、关闭退出、按钮灰化、批量调用和多选拖拽回归均通过。
