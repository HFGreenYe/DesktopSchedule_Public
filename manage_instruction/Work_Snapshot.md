# 架构改写前功能快照

本文档记录当前项目源码表现出的功能行为，用作后续架构重构的行为基线。

重构目标可以改变模块划分、文件组织、依赖注入方式和内部实现，但不应在无明确说明的情况下改变本文列出的用户可见行为、数据规则和跨视图同步逻辑。README 可能滞后，本文以当前源码为准。

## 1. 应用与窗口

- 应用入口是 `main.py`，创建 `QApplication` 后显示 `MainWindow`。
- 主窗口是 PyQt6 无边框窗口，默认使用固定宽度和最小高度。
- 主窗口默认宽度来自 `AppConfig.DEFAULT_WIDTH`，当前为 `324`。
- 主窗口最小高度当前为 `600`。
- 主窗口默认带 `WindowStaysOnTopHint`，也就是默认置顶。
- 主窗口、周视图、月视图、详情弹窗、待办看板都包含无边框、自绘背景、拖拽移动等行为。
- 项目针对 Windows 11 24H2 使用 `apply_24h2_border_fix` 修复透明窗口黑边、白边、圆角等问题。
- 主窗口背景为从 `AppConfig.COLOR_GRADIENT_START` 到 `AppConfig.COLOR_GRADIENT_END` 的纵向渐变。
- 多个二级页面和弹窗有独立的关闭按钮、悬挂按钮和窗口拖拽逻辑。
- 项目已有若干通用 UI 组件和辅助能力，包括 iOS 风格开关、数字滚轮、更多菜单、日程右键菜单、倒计时 tooltip、SVG 图标着色等。

## 2. 顶部 Header

- Header 显示当前时间，每秒刷新。
- Header 记录真实日期，检测跨天后发出 `midnight_rollover`，主窗口切回新的今天并刷新日视图。
- Header 显示日期文本，点击日期打开悬浮日历。
- 主窗口对日期文本安装滚轮事件过滤器：
  - 向上滚动切换到前一天。
  - 向下滚动切换到后一天。
  - 切换后刷新日视图，并更新 Header 日期文字。
- Header 显示天气图标和温度。
- 天气通过 IP 定位获取城市和经纬度，再调用和风天气当前天气接口。
- 天气配置来自 `.env` 中的 `QWEATHER_API_KEY` 和 `QWEATHER_API_HOST`。
- 天气初次启动后拉取一次，之后每 30 分钟刷新。
- Header 收到天气数据后更新主窗口天气，同时向周视图、月视图同步天气数据。
- Header、周视图、月视图的天气区域可显示城市、天气和温度相关 tooltip。
- Header 有搜索框，当前源码主要实现 UI，未看到实际搜索过滤行为。
- Header 工具按钮包含换肤、视图、添加、排序、筛选。
- 添加按钮有实际入口逻辑。
- 视图按钮会展开当前页面对应的视图选择器。
- 换肤、排序、筛选在主窗口中主要是占位或打印日志。
- 右上角更多菜单支持窗口置顶、显示农历、显示周数、打开待办看板、导出日程、全部日程等入口。
- 更多菜单中的打开待办看板有实际行为。
- 导出日程、全部日程当前是占位打印。
- 显示农历、显示周数当前回调为空或未完整接入。

## 3. 数据模型

数据库文件是项目根目录下的 `schedule.db`，ORM 使用 Peewee。

### Schedule

`Schedule` 表当前字段包括：

- `title`：标题。
- `description`：详情，可为空。
- `item_type`：类型，默认 `schedule`，待办使用 `todo`。
- `start_time`：开始时间，可为空。
- `end_time`：结束时间或截止时间，可为空。
- `reminder_time`：提醒时间，可为空。
- `is_alarm`：是否开启强提醒/闹钟。
- `alarm_duration`：闹钟持续模式。
- `priority`：优先级，当前约定 `0` 低、`1` 中、`2` 高。
- `repeat_rule`：重复规则。
- `category_id`：所属清单 ID，可为空。
- `theme_color`：字体/主题颜色字段，默认 `#FFFFFF`；当前源码未看到完整用户操作闭环。
- `status`：状态。
- `is_pinned`：是否置顶。
- `created_at`：创建或最近修改时间。
- `group_id`：重复日程组 ID。
- `sort_order`：手动排序权重。

### Category

`Category` 表当前字段包括：

- `name`：清单名称。
- `color`：清单主题色。
- `list_type`：清单类型，当前区分 `schedule` 和 `todo`。
- `is_deleted`：是否软删除。
- `created_at`：创建时间。
- `sort_order`：排序权重。

### 状态约定

- `status == 0`：未完成。
- `status == 1`：已完成。
- `status == 2`：隐藏。

### 启动迁移

- 应用启动时自动连接数据库并创建表。
- 启动时自动检查并迁移 `group_id`、`sort_order`、`list_type` 等字段。
- 旧数据补 `sort_order` 时，使用创建时间时间戳作为初始排序权重。
- 当前 `_migrate_db` 中 `categories.list_type` 分支存在 `migrator` 作用域风险：如果前面的 schedule 字段迁移分支都未执行，该分支可能引用未定义的 `migrator`。

## 4. 新建日程与待办

- 添加页支持两种模式：日程、待办。
- 从主日视图进入添加页时默认是日程模式。
- 从主待办页进入添加页时默认是待办模式。
- 添加页可以在日程/待办模式间切换。
- 标题不能为空。
- 日程模式必须设置时间，否则禁止保存，并显示提示。
- 待办模式不要求设置时间。
- 添加页详情输入区可展开/收起。
- 详情文本最大 150 字，超出会截断。
- 可选择优先级。
- 添加页的信息摘要行支持点击优先级并在低/中/高之间循环切换。
- 可选择重复规则。
- 可选择所属清单。
- 添加页和周视图添加页均有“设置字体”图标按钮，当前主要是 UI 入口，未看到实际字体或颜色选择流程。
- 日程模式显示时间、提醒、清单等信息。
- 待办模式隐藏时间和提醒入口，显示优先级和清单。
- 保存成功后写入数据库，触发刷新，并返回来源页。

## 5. 时间选择

- 时间选择页支持选择日期，日历最小日期是今天。
- 可只设置结束时间，也可开启开始时间。
- 开启开始时间后显示开始时间滚轮和快捷时长按钮。
- 快捷时长包括 10、15、30、45、60、90、120、180 分钟。
- 今天的开始时间和结束时间不能早于当前时间。
- 确认时再次校验开始时间和结束时间不能早于当前时间。
- 从添加页进入时，确认后回填添加页。
- 从详情编辑进入时，确认后更新数据库。
- 修改重复日程时间时，会询问修改所有/仅修改本次/取消。

## 6. 提醒与闹钟

- 提醒选择页基于目标时间设置提醒。
- 目标时间通常为开始时间，若无开始时间则使用结束时间。
- 支持快捷设置为目标时间前若干分钟。
- 若快捷提醒落到目标日期前一天，会自动打开“前一天”开关。
- 提醒时间不能早于当前时间。
- 提醒时间不能晚于目标时间。
- 可开启闹钟模式。
- 闹钟模式开启后显示持续模式选项。
- 闹钟持续模式包括 1 分钟、3 分钟、直到手动关闭。
- 确认后返回添加页或编辑保存。
- 主窗口每秒扫描所有日程提醒。
- 当前时间处于提醒时间之后 60 秒内，且该日程本次运行尚未触发过时，弹出提醒窗口。
- 触发过的日程 ID 会记录在内存集合，避免同次运行重复弹出。
- 提醒弹窗会显示目标时间倒计时；目标时间已过时显示“已开始”，无目标时间时显示 `00:00:00`。
- 提醒弹窗使用 `FramelessWindowHint | WindowStaysOnTopHint | Tool`，固定大小为 `320x180`，右上角关闭按钮绝对定位到窗口右上角。
- 如果 `is_alarm` 为真，会循环播放系统声音 `SystemHand`。
- 提醒弹窗的“关闭闹钟”、顶部关闭按钮和“知道了”关闭流程都会通过 `winsound.PlaySound(None, winsound.SND_PURGE)` 停止闹钟声音。

## 7. 重复日程

- 重复规则目前包括不重复、每天、每周、每月。
- 新建重复日程时会批量生成未来实例，并赋予同一个 `group_id`。
- 每天生成后续 365 条。
- 每周生成后续 52 条。
- 每月生成后续 12 条。
- 月重复通过 `_add_months` 处理月底日期溢出。
- 修改重复日程时支持两种影响范围：
  - 仅修改本次：当前项脱离原循环组，`group_id` 置空。
  - 修改未来/所有：删除原循环组未来项，再按新规则重建未来实例。
- 修改重复规则时，详情弹窗会提示保存后的影响。
- 从非重复日程改为重复日程时，会创建新的 `group_id` 并生成未来实例。
- 改为不重复时，当前日程会脱离循环组。

## 8. 日视图 Dashboard

- 日视图按当前选中日期加载日程。
- 当前日期默认是今天。
- 日历选择或 Header 滚轮切换会改变当前日期。
- 过滤隐藏项，即不显示 `status == 2`。
- 日视图不显示 `item_type == todo` 的待办项。
- 没有日程时显示空状态提示。
- 每 30 秒自动刷新。
- 卡片显示标题、时间、提醒状态、重复状态和优先级颜色点。
- 优先级颜色约定：
  - 高：红色。
  - 中：黄色。
  - 低：绿色。
- 已完成日程标题显示删除线，文字变淡。
- 过期未完成日程使用灰色背景和红色时间。
- 置顶日程有更明显边框或背景。
- 卡片右键菜单支持：
  - 完成日程。
  - 撤销完成。
  - 已完成后隐藏。
  - 置顶/取消置顶。
  - 删除。
- 点击卡片打开详情弹窗。
- 同一个日程已有详情弹窗时，再点击会激活已有弹窗。
- 卡片支持拖拽排序。
- 拖拽排序只在相同置顶状态和相同完成状态组内调整。
- 排序保存到 `sort_order`。
- 卡片悬停时会显示倒计时提示。

## 9. 详情弹窗

- 详情弹窗可从日视图、周视图、月视图、主待办页、独立待办看板打开。
- 弹窗显示标题、详情、时间、提醒、清单、创建/修改时间、优先级、重复规则。
- 待办类型隐藏时间、提醒、重复规则等日程专属信息。
- 从待办看板打开时，顶部会显示待办来源图标。
- 双击标题进入编辑，回车或失焦保存。
- 双击详情进入编辑，失焦保存。
- 双击时间发出编辑时间请求。
- 双击提醒发出编辑提醒请求。
- 双击清单发出编辑清单请求。
- 双击优先级显示下拉框，选择后立即保存。
- 双击重复规则显示下拉框，选择后按重复日程规则保存。
- 标题、详情、优先级、重复规则保存时会更新 `created_at`，当前代码把它作为最近修改时间展示。
- 弹窗自身支持置顶/取消置顶。
- 弹窗修改后会触发来源视图和相关视图刷新。

## 10. 清单与分类

- 清单按 `list_type` 区分日程清单和待办清单。
- 清单选择页根据当前条目类型加载对应清单。
- 可新建清单，新建后自动选中新清单。
- 清单卡片显示三位编号和名称，例如 `#001 名称`。
- 点击清单可选中；再次点击同一清单可取消选中。
- 清单右键可删除。
- 删除清单前会检查该清单关联项状态：
  - 空清单：直接硬删除。
  - 有有效未完成项：禁止删除。
  - 只有历史项：二次确认后软删除。
- 软删除清单仍可能被历史项引用，详情中会显示已删除状态。

## 11. 悬浮日历

- 点击 Header 日期打开悬浮日历。
- 悬浮日历大小约 300x260。
- 日历以周一为每周第一天。
- 有日程的日期底部显示小圆点。
- 日期状态颜色根据当天日程完成/逾期/未完成状态计算。
- 点击日期后关闭日历，并通知主窗口切换日视图日期。
- 日历导航栏可拖动整个弹窗。
- 双击导航栏可在深色/浅色主题之间切换。
- 今天会高亮显示。
- 跨天后会更新今天高亮。

## 12. 周视图

- 周视图是独立无边框窗口，大小约 680x400。
- 从主视图切换到周视图时，主窗口隐藏，周视图在主窗口附近居中显示。
- 从周视图切回其他视图时，主窗口在周视图当前位置附近居中恢复。
- 周视图显示当前周七天。
- 支持上一周、下一周。
- 周视图支持鼠标滚轮切换上一周/下一周。
- 周视图日期块显示农历日期、公历节日、农历节日和除夕等信息。
- 周视图日期块悬浮提示同时显示阳历日期和完整农历日期。
- 点击周视图日期块会选中该日并刷新该周数据。
- 顶部显示时间、周数、天气、工具按钮、搜索框和视图选择器。
- 工具按钮包括换肤、视图、添加、排序、筛选。
- 周视图里的添加按钮进入周视图自己的添加流程。
- 周视图当前选中日期早于今天时禁止添加日程，并显示 toast。
- 周视图内复用专属的添加页、时间选择页、提醒页和清单选择页。
- 周视图每一天一列展示日程。
- 点击日程卡片会打开详情弹窗。
- 右键日程卡片可完成/撤销完成/隐藏/置顶/删除。
- 支持拖拽排序。
- 支持跨列拖拽，目标列会影响日程日期。
- 周视图跨列拖拽到过去日期时会被拒绝，并刷新让卡片回到原位。
- 周视图编辑时间、提醒、清单时走周视图自己的路由。
- 周视图可切换到日视图、月视图、待办视图。
- 周视图支持挂起为周视图专属悬浮条。

## 13. 月视图

- 月视图是独立无边框窗口，大小约 720x480。
- 从主视图切换到月视图时，主窗口隐藏，月视图在主窗口附近居中显示。
- 从月视图切回其他视图时，主窗口在月视图当前位置附近居中恢复。
- 左侧显示时间、天气、月份切换、工具按钮、搜索框和内联添加区域。
- 右侧显示大月历。
- 支持上一月、下一月。
- 月历单元格自绘：
  - 本月工作日白色。
  - 本月周末红色。
  - 非本月日期灰色半透明。
  - 选中和悬停有叠层。
- 点击日期会通知主窗口跳转到该日期的日视图。
- 月视图可内联快速添加日程。
- 月视图内联添加可填写标题和描述；标题为空时禁止保存并显示 toast。
- 月视图内联添加默认保存到选中日期的 00:00，开始时间和结束时间相同。
- 月视图选中日期早于今天时禁止打开内联添加，并显示 toast。
- 月视图支持视图切换。
- 月视图支持挂起为月视图专属悬浮条。
- 月视图搜索、排序、筛选当前主要是 UI，未看到完整过滤逻辑。

## 14. 主待办视图

- 主窗口内有待办页。
- 从视图选择器可切换到待办页。
- 待办页只显示 `item_type == todo` 的项目。
- 过滤隐藏项 `status == 2`。
- 无待办时显示空状态提示。
- 每 30 秒自动刷新。
- 排序规则：
  - 置顶优先。
  - 未完成优先。
  - 已完成靠后。
  - 同组内 `sort_order` 越大越靠前。
- 待办卡片显示标题、优先级颜色点和清单标签。
- 已完成待办标题显示删除线。
- 右键菜单支持完成/撤销完成、隐藏、置顶/取消置顶、删除。
- 点击待办打开详情弹窗。
- 支持拖拽排序，排序结果写入 `sort_order`。
- 从待办页进入添加页时默认创建待办。

## 15. 独立待办看板

- 通过更多菜单中的待办显示入口打开或隐藏独立待办看板。
- 待办看板是独立无边框小窗口，默认显示在主窗口右侧。
- 看板支持拖拽移动。
- 看板支持窗口置顶。
- 看板有便签视图和文件夹视图。
- 便签视图显示未完成待办。
- 文件夹视图按待办清单展示文件夹。
- 可进入某个文件夹，只显示该清单下的待办。
- 可从文件夹内部返回文件夹列表。
- 文件夹视图包含固定的“未分类待办”文件夹，该固定文件夹不可拖动、不可删除。
- 待办文件夹卡片支持拖拽排序，排序结果写入清单的 `sort_order`。
- 看板支持内联添加待办。
- 在文件夹内部添加待办时，默认选中当前文件夹对应清单。
- 看板内添加待办也可打开清单选择页。
- 看板支持清单管理视图，可新增和删除待办清单。
- 文件夹删除规则与清单删除规则一致：
  - 有有效待办：禁止删除。
  - 有历史待办：二次确认后软删除。
  - 空文件夹：硬删除。
- 看板支持按优先级排序：
  - 高优先级在前。
  - 同优先级按创建时间从早到晚。
  - 排序结果写入 `sort_order`。
- 看板卡片支持完成、置顶、删除。
- 看板卡片可打开详情弹窗。
- 从看板打开的详情弹窗会移动到看板旁边。
- 看板每 1 秒刷新一次。
- 看板使用数据签名避免没有变化时重复重绘。
- 看板操作会通知主待办页、日视图、周视图等刷新。

## 16. 挂起模式

- 主窗口支持挂起为 `SuspendWindow`。
- 周视图支持挂起为 `SuspendWindowWeek`。
- 月视图支持挂起为 `SuspendWindowMonth`。
- 挂起时隐藏原窗口，显示对应悬浮条。
- 从悬浮条恢复时，隐藏悬浮条并显示原窗口。
- 恢复时大致保留窗口位置。
- 主窗口从挂起恢复后会重设默认宽度和高度。

## 17. 跨视图刷新

- 日视图、主待办页、周视图、月视图、独立待办看板之间通过信号和直接方法调用同步刷新。
- 新增、编辑、删除、完成、隐藏、置顶、排序后会触发相关视图刷新。
- 部分刷新流程会同步更新已经打开的详情弹窗数据。
- 主窗口维护日视图详情弹窗列表，避免同一条日程重复打开多个详情弹窗。
- 独立待办看板会尝试同步主窗口中已打开的详情弹窗。

## 19. 重构验收建议

重构后至少按以下路径人工验收：

1. 启动应用，确认主窗口显示、天气失败时不崩溃、时间每秒刷新。
2. 新建一个普通日程，设置时间、提醒、清单、优先级，确认日视图显示正确。
3. 新建一个待办，确认主待办页和独立待办看板显示正确。
4. 修改日程标题、详情、时间、提醒、清单、优先级，确认数据库和各视图同步。
5. 创建每天重复日程，确认生成未来实例；修改本次和修改未来分别符合预期。
6. 完成、撤销完成、隐藏、置顶、删除日程和待办，确认各视图同步刷新。
7. 在日视图和待办视图拖拽排序，确认重启后顺序仍保持。
8. 切换周视图、月视图、待办视图，再切回日视图，确认窗口位置和数据刷新正常。
9. 打开悬浮日历选择日期，确认日视图跳转正确。
10. 打开独立待办看板，测试便签视图、文件夹视图、文件夹内添加、清单删除规则。
11. 测试主窗口、周视图、月视图挂起和恢复。
12. 测试提醒时间到达时弹窗出现，闹钟模式播放系统声音。
