# DesktopSchedule 最终构想与实施路线

> 最后更新：2026-07-19。本文件记录项目最终形态、阶段归档和后续实施顺序；已完成阶段不再反复重构，后续优先完成功能闭环。

## 最终产品定位

DesktopSchedule 最终定位为一款面向 Windows 桌面的本地优先日程管理工具：

- 以日程、待办、纪念日期为核心数据，统一覆盖日、周、月和待办看板。
- 日、周、月视图同时保留卡片模式与课表模式，兼顾快速浏览和精确时间安排。
- 无网络时保持全部核心功能可用；云同步作为可选能力，不改变本地数据库的主数据地位。
- 提供 Markdown、PDF、PNG 导出，并支持默认背景、自定义图片、裁剪、字体、颜色和尺寸记忆。
- 保持轻量、透明、低干扰的桌面面板体验，最终统一主窗口、独立窗口和各类弹窗的设计语言。

## 后续任务顺序

### 1. 多选与批量操作闭环

- 将现有“多选”菜单入口绑定为真实选择状态。
- 日、周卡片支持批量完成、撤销完成和删除。
- 纪念日期支持批量删除。
- 明确空白区、卡片区和课表模式的入口边界，避免与拖拽、右键卡片菜单冲突。
- 增加退出多选、全选、选择计数和危险操作二次确认。

### 2. 云同步完整闭环

- 坚持本地优先，先定义同步数据范围、数据版本和迁移策略。
- 为日程、分类、待办和纪念日期设计稳定的跨设备标识、更新时间和删除墓碑。
- 实现增量推送、增量拉取、离线队列、重试和幂等处理。
- 明确冲突策略，默认避免静默覆盖，并提供可理解的冲突提示或恢复入口。
- 对令牌和账号信息使用系统安全存储，不把凭据写入数据库、日志或仓库。
- 为工具栏同步按钮补充未同步、同步中、成功、失败和离线状态。
- 增加首次同步、本地恢复、换机、断网、重复同步和冲突场景测试。

### 3. 业务规则补全

- 实装 `每年` / `yearly` 重复日程。
- 评估并补充重复实例来源字段，减少当前 `group_id` 语义承担过多职责的问题。
- 完成主题、字体和颜色设置的全局用户操作流程。
- 评估四象限视图及其与现有分类、重要性规则的关系，再决定是否实装。

### 4. 稳定性与测试

- 为重复规则、查询筛选、排序、导出裁剪、纪念日期和同步合并增加服务层测试。
- 增加数据库迁移、异常恢复和关键 UI 状态的回归检查。
- 清理仍直接依赖 `DatabaseManager` 的新增业务路径，继续收敛到 Service / Repository。

### 5. 最终 UI 审美统一

该阶段必须放在云同步及其他功能完整实装之后：

- 保持“克制、清透、低干扰的桌面日程玻璃面板”方向，不推翻现有辨识度。
- 统一字体、字号、间距、圆角、透明度、阴影、控件尺寸和高 DPI 可读性。
- 减少普通区域中过多的白色描边和多层半透明叠加，改用层级、留白和轻阴影分区。
- 统一主界面和弹窗的标题栏高度、图标对齐、按钮热区及内容边距。
- 优先复查纪念日期、导出设置、日期弹窗和二级菜单等信息密集区域。
- 补充克制的点击、弹出和拖拽反馈，并兼顾减少动态效果和减少透明度设置。

## 完成标准

项目进入最终稳定版前应满足：

- 核心功能在离线状态下完整可用，云同步失败不会阻断本地操作。
- 多设备同步具备可恢复、可追踪和不静默丢失数据的基本保障。
- 日、周、月、待办、纪念日期和导出功能具有稳定的一致交互。
- 关键业务规则有自动化测试，数据库升级有明确迁移路径。
- 最终 UI 调整完成后，对主要窗口、弹窗、不同主题和高 DPI 环境统一回归。
